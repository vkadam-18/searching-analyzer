Binary Search







How Binary Search Works:

  1. Start with two pointers representing the start and end of the array.
  2. Find the middle element of the array.
  3. If the middle element is the target, return its index.
  4. If the target is less than the middle element, repeat the process on the left sub-array.
  5. If the target is greater, repeat on the right sub-array.
  6. If the target is not found, return -1.

Example:

Array: [2, 3, 4, 10, 40]
Target: 10
Steps:
- Middle element is 4 (index 2).
- Target is greater than 4, search in the right sub-array: [10, 40].
- Middle element is 10 (index 3) - match found.

Time Complexity:

  • Best Case: O(1) – When the target is found at the middle element.
  • Average Case: O(log n) – When the target is located in the middle of the array after several splits.
  • Worst Case: O(log n) – When the target is located at one of the extremes of the array or not found.

Advantages:

  • Efficient for large sorted datasets.
  • Time complexity is O(log n).

Disadvantages:

  • Only works on sorted arrays.
  • More complex to implement compared to linear search.
