Ternary Search







How Ternary Search Works:

  1. Start with two pointers representing the start and end of the array.
  2. Divide the array into three equal parts by finding two midpoints.
  3. Compare the target value with the two midpoints.
  4. If it matches either midpoint, return that index.
  5. If the target is less than the first midpoint, repeat on the left sub-array.
  6. If it's between the midpoints, repeat on the middle sub-array.
  7. If greater than the second midpoint, repeat on the right sub-array.
  8. If the target is not found, return -1.

Example:

Array: [1, 3, 5, 7, 9, 11, 13]
Target: 7
Steps:
- Divide into [1, 3, 5] | [7, 9, 11, 13]
- Midpoints are 5 (index 2) and 9 (index 4).
- Target is between 5 and 9, so search in the middle sub-array: [7].
- Match found at index 3.

Time Complexity:

  • Best Case: O(1) – When the target is found at one of the split points.
  • Average Case: O(log₃ n) – When the target is located after several ternary splits.
  • Worst Case: O(log₃ n) – When the target is at one of the extremes of the array or not found.

Advantages:

  • Faster than binary search for some datasets.
  • Useful for searching in a sorted array.

Disadvantages:

  • More complex to implement than binary search.
  • Only works on sorted arrays.
  • Time complexity is O(log3 n).
